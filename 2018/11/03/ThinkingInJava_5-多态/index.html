方法绑定

将一个方法调用同一个方法主体关联起来被称作绑定，当绑定发生在程序运行之前时（如果有的话，由编译器和连接器负责，C就是一种前期绑定的语言）称为前期绑定（early binding）。 在运行时，根据对象的类型来决定运行哪个方法称为后期绑定（late binding)，后期绑定也被称为动态绑定（dynamic binding）或运行时绑定(run-time binding)。

Java 中除了static和final方法之外，其他所有的方法都是属于后期绑定。

下面讲一个高级的示例，是方法绑定中极易混淆的一点：

创建一个导出类的对象，并将它向上造型，调用从基类继承的方法f1()，f1()方法内部却调用了导出类的f2()，这是绑定的一个绝佳的示例。

然而，多态与绑定并非都如上例所示，只有普通的方法调用是多态的，接下来介绍两种特殊的情况。

  • 缺陷：直接访问某个域

在上例中为Super.field和Sub.field分配了不同的存储空间，这样实际上Sub包含两个称为field的域：自己的和从super处得到的。当引用Sub中的field时，是继承自父类的field，在引用Sub中的getField方法时调用的是子类的field，因此为了调用Super.field，必须显式地注明super.field。在实践中为了避免这样的误解，通常会将所有的域都设置成private，因此不能直接访问它们，其副作用是只能调用方法来访问。另外，不要对基类中的域和导出类中的域赋予同样的名字。

  • 缺陷：静态方法的行为不具有多态性

结果有点令人惊讶，因为在父类构造方法中调用了draw()方法，而此方法在子类中被覆盖，所以在父类的构造方法调用时，会动态绑定到子类的draw()方法，而此时子类的成员变量还未定义。所以编写构造器有一条有效的原则：“用尽可能简单的方法使对象进入正常的状态；如果可以的话，避免调用其他方法”。在构造器中，唯一安全能够调用的方法是基类中的final方法（包括private方法)，因为这些方法无法被覆盖。

协同返回类型

先复习一下重要概念：

  • 重载：同一个方法的参数情况不一样，指的是方法的重载，一般是在同一个类中，重载跟返回值没关系。
  • 重写：多态中，指的是子类和父类方法名称、返回值和参数完全一致，指的是方法的覆盖（重写）。

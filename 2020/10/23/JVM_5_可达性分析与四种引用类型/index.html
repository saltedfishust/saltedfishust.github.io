可达性分析与四种引用类型

发表于 2020-10-23 更新于 2021-03-05 分类于 JVM
本文字数： 8.1k 阅读时长 ≈ 15 分钟

四种引用

常见的准确来说是五种引用，我们平时用的最多的类似“Object j = new Object()”就是强引用，不同于强引用，其他的所有引用类型都有特定的类实现（括号里类），例如软引用是SoftReference类……其中这些非强引用都可以（或者必须）配合引用队列来使用，引用队列的好处是可以释放这些引用对象自身占用的内存空间。用终结器引用回收垃圾效率极低，而且运行代价高昂，用的不多也不建议用，了解即可。

强引用
  • 只要 GC Roots 到对象的强引用还存在，对象就不会被垃圾回收，换句话说，只有所有 GC Roots 对象都不通过“强引用”引用该对象，该对象才能被垃圾回收
软引用（SoftReference）
  • 仅有软引用引用该对象时，在第一次垃圾回收后，内存仍不足时会再次触发第二次垃圾回收，回收软引用对象，即内存不够会被垃圾回收
  • 可以配合引用队列来释放软引用自身
弱引用（WeakReference）
  • 仅有弱引用引用该对象时，在第一次垃圾回收时，无论内存是否充足，都会回收弱引用对象
  • 可以配合引用队列来释放弱引用自身
  • 适合弱引用的场景，往往可以搭配WeakHashMap来实现

    注意：上面的“第一次、第二次垃圾回收”指的是清理到它们才算一次，即如果对象在老年代而发生的是新生代GC则不算是清理到它们。

虚引用（PhantomReference）
  • 为一个对象设置虚引用关联的唯一目的就是能在这个对象被收集器回收时收到一个系统通知
  • 必须配合引用队列使用，主要配合 ByteBuffer 使用（见直接内存那一节），被引用对象byteBuffer被回收时系统会收到一个通知，然后将虚引用入队， 由 Reference Handler 线程调用虚引用相关方法释放直接内存
终结器引用（FinalReference）
  • 无需手动编码，但其内部必须配合引用队列使用，在垃圾回收时，当对象重写了finalize()方法，终结器引用入队（被引用对象暂时没有被回收），再由 Finalizer 线程通过终结器引用找到被引用对象并调用它的 finalize 方法，第二次 GC 时才能回收被引用对象
